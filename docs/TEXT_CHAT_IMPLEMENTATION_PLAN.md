# テキストチャット機能追加計画

## 🎯 概要

既存の音声中心Virtual Human Chatアプリケーションに、テキスト入力・出力機能を追加。音声とテキストが両立できるハイブリッド対話システムを構築し、レスポンスも音声・文字両方で表示可能にする。既存機能を破綻させずに段階的実装を行う。

## 📋 機能要件

- ✅ テキスト入力フィールドの追加（音声と併用可能）
- ✅ AIレスポンスのテキスト表示強化（現在の[音声応答]から実際のテキストへ）  
- ✅ 音声・テキスト混在の会話履歴表示
- ✅ 音声録音中でもテキスト入力可能なUI設計
- ✅ Gemini Live APIのテキストレスポンス活用
- ✅ 既存の音声機能との完全な互換性維持
- ✅ レスポンシブUI対応（モバイル・デスクトップ）
- ✅ アクセシビリティ対応の維持・強化

## 🎨 UI設計原則

- **音声優先維持**: 既存の音声UXを維持しつつ、テキストは補完的に配置
- **明確な区別**: ChatHistory.tsxでメッセージタイプ（audio/text/hybrid）を明確に区別
- **統合レイアウト**: ControlPanel下部にテキスト入力エリアを追加（折りたたみ可能）
- **マルチモーダル**: 音声録音中のテキスト入力制限なし（マルチモーダル対話）
- **一貫性**: shadcn/ui Input/Textarea コンポーネントで一貫性保持
- **レスポンシブ**: モバイル対応（スマートフォンでもテキスト・音声両方が使いやすい）
- **アクセシビリティ**: ARIA labels、キーボードナビゲーション維持

## 🔧 技術的考慮事項

### 型定義の拡張
```typescript
interface ChatMessage {
  role: 'user' | 'assistant'
  text: string
  messageType: 'audio' | 'text' | 'hybrid'  // 新規追加
  audioUrl?: string
  timestamp: number
}
```

### API統合
- **Gemini Live API**: response_modalities を ['AUDIO', 'TEXT'] に拡張
- **新規エンドポイント**: POST /api/gemini/text（RESTful設計）
- **パフォーマンス**: テキスト送信時は音声処理チェーンをバイパス

### 状態管理
- **useChatStore**: sendTextMessage action追加、addMessage更新
- **独立状態**: 音声・テキスト入力状態の独立管理
- **エラーハンドリング**: 音声失敗時のテキストフォールバック機能
